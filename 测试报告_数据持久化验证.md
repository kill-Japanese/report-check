## 一、问题排查

### 问题描述
新增的项目在重新登录后消失。

### 根本原因（3个时序/逻辑问题）

| # | 问题 | 影响 |
|---|------|------|
| 1 | `refreshRawData()` 覆盖 `RAW_DATA.allProjects`，但 `mergeLocalNewProjects()` 在它之前同步执行 | 还未写入Excel的本地新增项目被覆盖丢失 |
| 2 | `collabSyncToServer()` 同步成功后无条件清空 `newProjects`（localStorage） | 即使项目未真正写入Excel也被清除，刷新后无法恢复 |
| 3 | `collabLoadData()` 用服务器的空 `newProjects` 覆盖本地 localStorage | 本地待确认的新增项目被服务器空数据覆盖 |

---

## 二、修复内容

### 修复1：`refreshRawData()` 后重新合并 localStorage 项目
```javascript
// 刷新后重新合并 localStorage 中的新增项目
const merged = mergeLocalNewProjects();
```

### 修复2：`collabSyncToServer()` 只移除已确认的项目
```javascript
// 只移除服务器中已存在的项目，未确认的保留待重试
const serverIds = new Set(RAW_DATA.allProjects.map(p => p.id));
const pendingProjects = newProjects.filter(np => !serverIds.has(np.id));
```

### 修复3：`collabLoadData()` 不覆盖本地 `newProjects`
```javascript
// 不直接用服务器的 newProjects 覆盖本地的
// localStorage.setItem('newProjects', JSON.stringify(newProjects));  // 已注释
```

### 相关修复（之前已完成）
- `resetAll()` 不再清空归档和删除状态
- 新增 `/api/projects` 端点获取最新Excel数据
- `read_excel_projects()` 日期格式化解决JSON序列化问题
- `git_push()` 先pull后push解决本地落后问题
