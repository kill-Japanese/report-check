## 🏆 方案一：Oracle Cloud 永久免费 VPS（最推荐）

### 为什么选这个？
- **真正永久免费**：4 核 ARM CPU + 24GB 内存 + 200GB 存储，永远不收费 [$TRAE_REF](https://www.oracle.com/cn/cloud/free/)
- **独立公网 IP**：地址固定不变
- **功能 100% 完整**：定时任务、邮件服务、协作服务全部可用
- **可同时部署其他应用**：资源非常充裕

### 第一步：注册 Oracle Cloud 账号

1. 访问 https://www.oracle.com/cn/cloud/free/
2. 点击「开始免费试用」，用邮箱注册（建议 Gmail/Outlook）
3. 绑定信用卡（仅验证，不会扣费，可选虚拟信用卡）
4. 注册成功后进入控制台

### 第二步：创建免费 VPS 实例

1. 控制台搜索「Compute」→「Instances」→「Create instance」
2. **配置选择**：
   - Name: `report-server`
   - Image: **Ubuntu 22.04**（或 24.04）
   - Shape: **VM.Standard.A1.Flex**（ARM 架构，免费）
     - OCPU: **4**（最多免费 4 核）
     - Memory: **24 GB**（最多免费 24GB）
   - Boot volume: **200 GB**（最大免费）
3. 下载 SSH 私钥（保存好！）
4. 点击「Create」，等待 2-3 分钟

### 第三步：开放防火墙端口

1. 实例详情页 → 点击「Virtual cloud network」
2. 点击「Security Lists」→ 默认安全组 →「Add Ingress Rules」
3. 添加以下规则：
   ```
   端口 80    (HTTP)    源: 0.0.0.0/0
   端口 443   (HTTPS)   源: 0.0.0.0/0
   端口 8080  (协作)    源: 0.0.0.0/0
   端口 8765  (邮件)    源: 0.0.0.0/0
   ```
4. 同时在 Oracle 防火墙放行：
   ```bash
   sudo iptables -I INPUT -p tcp --dport 80 -j ACCEPT
   sudo iptables -I INPUT -p tcp --dport 443 -j ACCEPT
   sudo iptables -I INPUT -p tcp --dport 8080 -j ACCEPT
   sudo iptables -I INPUT -p tcp --dport 8765 -j ACCEPT
   sudo netfilter-persistent save
   ```

### 第四步：连接服务器并部署

```bash
# 用 SSH 连接（Windows 用 PowerShell，Mac/Linux 用终端）
ssh -i /path/to/your-key.key ubuntu@你的服务器IP

# 一键部署（复制粘贴下面所有内容）
sudo apt update && sudo apt install -y python3-pip python3-venv git cron nginx

# 创建项目目录
mkdir -p ~/report && cd ~/report

# 上传你的项目文件到这里（可以用 scp 或 git）
# 示例：如果用 GitHub
# git clone https://github.com/你的用户名/你的仓库.git .

# 安装依赖
pip3 install pandas openpyxl --break-system-packages

# 设置定时任务（每天北京时间 8:20 = UTC 0:20）
echo '20 0 * * * cd /home/ubuntu/report && python3 更新点检表.py >> /home/ubuntu/report/cron.log 2>&1' | crontab -

# 启动协作服务（后台运行）
nohup python3 协作服务器.py 8080 > /home/ubuntu/report/collab.log 2>&1 &

# 启动邮件服务（如果需要）
# nohup python3 mail_server.py > /home/ubuntu/report/mail.log 2>&1 &

echo "✅ 部署完成！"
echo "   协作地址: http://你的服务器IP:8080"
echo "   定时任务: 每天 8:20 自动生成报表"
```

### 第五步（可选）：配置 Nginx + 免费域名

```bash
# 安装 Nginx（已装就跳过）
sudo apt install -y nginx

# 配置反向代理
sudo tee /etc/nginx/sites-available/report <<'EOF'
server {
    listen 80;
    server_name 你的域名.com;  # 没有域名就写服务器IP

    location / {
        proxy_pass http://127.0.0.1:8080;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
    }
}
EOF

sudo ln -sf /etc/nginx/sites-available/report /etc/nginx/sites-enabled/
sudo nginx -t && sudo systemctl restart nginx

# 免费 HTTPS（Let's Encrypt）
sudo apt install -y certbot python3-certbot-nginx
sudo certbot --nginx -d 你的域名.com
```

> **免费域名申请**：去 https://www.freenom.com 或 https://nic.ua 申请免费顶级域名

## 🚀 方案二：本地电脑 + Cloudflare Tunnel（最简单）

### 为什么选这个？
- **零成本，10 分钟搞定**
- **固定公网地址**（Cloudflare 免费提供）
- **无需公网 IP，无需开端口**
- **功能完整**：所有功能都能跑

> 唯一要求：电脑需要常开（或每天 8:20 前开机）

### 第一步：安装 Cloudflare Tunnel

1. 访问 https://www.cloudflare.com 注册免费账号
2. 下载 `cloudflared` 工具：
   ```bash
   # Windows (PowerShell)
   winget install Cloudflare.cloudflared

   # Mac
   brew install cloudflared

   # Linux
   wget https://github.com/cloudflare/cloudflared/releases/latest/download/cloudflared-linux-amd64
   chmod +x cloudflared-linux-amd64 && sudo mv cloudflared-linux-amd64 /usr/local/bin/cloudflared
   ```

### 第二步：登录并创建隧道

```bash
# 登录 Cloudflare（会打开浏览器授权）
cloudflared tunnel login

# 创建隧道
cloudflared tunnel create report-server

# 记下输出的隧道 ID，例如：a1b2c3d4-xxxx-xxxx-xxxx-xxxxxxxxxxxx

# 配置隧道
cloudflared tunnel route dns report-server report  # 配置子域名 report.你的域名.com
# 或者用免费的 trycloudflare.com 地址（无需域名）
```

### 第三步：创建配置文件

在项目目录下创建 `config.yml`：

```yaml
tunnel: 你的隧道ID
credentials-file: /home/用户名/.cloudflared/你的隧道ID.json

ingress:
  - hostname: report.你的域名.com    # 或你拿到的 trycloudflare.com 地址
    service: http://localhost:8080
  - service: http_status:404
```

### 第四步：启动所有服务

**Windows**（创建 `启动所有服务.bat`）：
```batch
@echo off
cd /d %~dp0

echo 正在启动协作服务器...
start "协作服务器" python 协作服务器.py 8080

echo 正在启动 Cloudflare Tunnel...
start "Cloudflare" cloudflared tunnel --config config.yml run

echo 正在设置定时任务...
schtasks /create /tn "每日点检报表" /tr "python %cd%\更新点检表.py" /sc daily /st 08:20

echo ✅ 所有服务已启动！
pause
```

**Mac/Linux**（创建 `启动所有服务.sh`）：
```bash
#!/bin/bash
cd "$(dirname "$0")"

# 启动协作服务器
nohup python3 协作服务器.py 8080 > collab.log 2>&1 &
echo "✅ 协作服务器已启动"

# 启动 Cloudflare Tunnel
nohup cloudflared tunnel --config config.yml run > cloudflare.log 2>&1 &
echo "✅ Cloudflare Tunnel 已启动"

# 设置定时任务（每天 8:20）
(crontab -l 2>/dev/null; echo "20 8 * * * cd $(pwd) && python3 更新点检表.py >> cron.log 2>&1") | crontab -
echo "✅ 定时任务已设置"

echo ""
echo "🎉 所有服务启动完成！"
echo "   协作地址: 你的 Cloudflare 域名"
```

### 开机自启设置

**Windows**：把 `启动所有服务.bat` 放到：
```
C:\Users\你的用户名\AppData\Roaming\Microsoft\Windows\Start Menu\Programs\Startup\
```

**Mac**：系统设置 → 通用 → 登录项 → 添加脚本

**Linux**：用 systemd 服务（参见方案一）

## ☁️ 方案三：Render 免费云托管

### 为什么选这个？
- **永久免费**，无需信用卡
- **地址固定**：`你的应用.onrender.com`
- **无需管理服务器**
- **自动 HTTPS**

> 注意：免费版 15 分钟无访问会休眠，首次打开需要等几秒

### 第一步：准备代码仓库

1. 去 https://github.com 注册免费账号
2. 创建一个新仓库（Private 也可以）
3. 把你的项目文件上传到仓库

### 第二步：创建部署配置

在项目根目录创建以下文件：

**`requirements.txt`**：
```
pandas>=2.0.0
openpyxl>=3.1.0
```

**`render.yaml`**（可选，一键配置）：
```yaml
services:
  - type: web
    name: report-check
    env: python
    buildCommand: pip install -r requirements.txt
    startCommand: python 协作服务器.py 10000
    envVars:
      - key: PYTHON_VERSION
        value: 3.11.0
```

**`Procfile`**（Render 启动命令）：
```
web: python 协作服务器.py 10000
```

### 第三步：部署到 Render

1. 访问 https://render.com 用 GitHub 登录
2. 点击「New +」→「Web Service」
3. 选择你刚才的 GitHub 仓库
4. 配置：
   - **Name**: 随便起（会成为子域名）
   - **Region**: 选新加坡（Singapore，离国内近）
   - **Runtime**: Python 3
   - **Build Command**: `pip install -r requirements.txt`
   - **Start Command**: `python 协作服务器.py 10000`
5. 点击「Create Web Service」
6. 等待 2-5 分钟部署完成

### 第四步：设置定时任务（每天 8:20）

Render 免费版不支持内置 cron，但可以用免费的外部服务触发：

**方案 A：cron-job.org（推荐，完全免费）**

1. 访问 https://cron-job.org 注册
2. 创建新任务：
   - **URL**: `https://你的应用.onrender.com/api/trigger-report`
   - **Schedule**: `20 0 * * *`（UTC 0:20 = 北京时间 8:20）
   - **Method**: GET
3. 保存即可

需要在 `协作服务器.py` 中添加触发端点（见下文自动化脚本）。

**方案 B：GitHub Actions（免费，推荐）**

在你的仓库创建 `.github/workflows/daily-report.yml`：

```yaml
name: 每日报表生成
on:
  schedule:
    - cron: '20 0 * * *'  # UTC 0:20 = 北京时间 8:20
  workflow_dispatch:       # 允许手动触发

jobs:
  generate-report:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - uses: actions/setup-python@v5
        with:
          python-version: '3.11'
      - run: pip install pandas openpyxl
      - run: python 更新点检表.py
      - name: 上传报表
        uses: actions/upload-artifact@v4
        with:
          name: 点检报表
          path: |
            项目延期点检表.html
            项目延期点检表.xlsx
```

这样每天 8:20 会自动生成报表并保存到 GitHub Actions 中。

## 🐳 方案四：Koyeb 容器化部署

### 为什么选这个？
- **免费额度够用**：每月 5 美元免费额度，单实例基本够用
- **全球节点**：访问速度快
- **支持 Docker**：部署灵活

### 第一步：准备 Dockerfile

在项目根目录创建 `Dockerfile`：

```dockerfile
FROM python:3.11-slim

WORKDIR /app

# 安装依赖
COPY requirements.txt .
RUN pip install --no-cache-dir -r requirements.txt

# 复制项目文件
COPY . .

# 暴露端口
EXPOSE 8080

# 启动命令
CMD ["python", "协作服务器.py", "8080"]
```

创建 `.dockerignore`：
```
__pycache__/
*.pyc
.git/
.uploads/
协作数据.json
cron.log
*.log
```

### 第二步：部署到 Koyeb

1. 访问 https://www.koyeb.com 注册（用 GitHub 登录）
2. 点击「Create App」→「Create Web Service」
3. 选择「Docker」部署方式
4. 连接你的 GitHub 仓库
5. 配置：
   - **Instance size**: Nano（免费额度内）
   - **Scaling**: Min 1, Max 1
   - **Port**: 8080
6. 点击「Deploy」

### 第三步：定时任务

同方案三，使用 cron-job.org 或 GitHub Actions。

## 📧 邮件功能部署说明

所有方案的邮件功能有两种实现方式：

### 方式 A：本地邮件客户端（最简单，推荐）

HTML 页面已内置「打开邮件客户端」功能，点击后直接唤起 Outlook/Foxmail/Apple Mail，无需任何部署。✅

### 方式 B：自动邮件发送服务

如果需要点击按钮自动发邮件（不打开客户端）：

1. 配置 `email_config.py`：
   ```python
   SMTP_HOST = "smtp.qq.com"      # 或 smtp.163.com / smtp.gmail.com
   SMTP_PORT = 465
   SMTP_USER = "你的邮箱@qq.com"
   SMTP_PASSWORD = "你的SMTP授权码"  # 不是邮箱密码！
   ```

2. 启动邮件服务：
   ```bash
   python mail_server.py
   ```

3. 在云服务器上部署为后台服务（方案一适用）：
   ```bash
   # 创建 systemd 服务
   sudo tee /etc/systemd/system/mail-server.service <<'EOF'
   [Unit]
   Description=邮件发送服务
   After=network.target

   [Service]
   User=ubuntu
   WorkingDirectory=/home/ubuntu/report
   ExecStart=/usr/bin/python3 mail_server.py
   Restart=always

   [Install]
   WantedBy=multi-user.target
   EOF

   sudo systemctl enable --now mail-server
   ```

## 🔧 一键自动化部署脚本

以下脚本适用于 **方案一（Oracle Cloud）** 和 **任何 Ubuntu/Debian 服务器**。

创建 `deploy.sh`：

```bash
#!/bin/bash
# ============================================================
# 超声波户表点检系统 - 一键部署脚本
# 适用：Ubuntu 22.04+ / Debian 12+
# ============================================================

set -e

echo "============================================"
echo "🚀 超声波户表点检系统 - 一键部署"
echo "============================================"

# 1. 安装基础依赖
echo ""
echo "[1/6] 安装系统依赖..."
sudo apt update -qq
sudo apt install -y -qq python3-pip python3-venv git cron nginx curl
echo "✅ 系统依赖安装完成"

# 2. 安装 Python 依赖
echo ""
echo "[2/6] 安装 Python 依赖..."
pip3 install pandas openpyxl --break-system-packages -q
echo "✅ Python 依赖安装完成"

# 3. 创建数据目录
echo ""
echo "[3/6] 初始化目录..."
mkdir -p .uploads .trae
touch cron.log collab.log mail.log
echo "✅ 目录初始化完成"

# 4. 设置定时任务（每天北京时间 8:20）
echo ""
echo "[4/6] 设置定时任务..."
CRON_CMD="20 0 * * * cd $(pwd) && /usr/bin/python3 更新点检表.py >> $(pwd)/cron.log 2>&1"
(crontab -l 2>/dev/null | grep -v "更新点检表"; echo "$CRON_CMD") | crontab -
echo "✅ 定时任务已设置（每天 8:20 自动生成报表）"

# 5. 配置协作服务为 systemd 服务
echo ""
echo "[5/6] 配置协作服务..."
sudo tee /etc/systemd/system/report-collab.service > /dev/null <<EOF
[Unit]
Description=项目点检表协作服务器
After=network.target

[Service]
User=$(whoami)
WorkingDirectory=$(pwd)
ExecStart=/usr/bin/python3 $(pwd)/协作服务器.py 8080
Restart=always
RestartSec=5

[Install]
WantedBy=multi-user.target
EOF

sudo systemctl daemon-reload
sudo systemctl enable --now report-collab
echo "✅ 协作服务已启动（端口 8080）"

# 6. 配置 Nginx 反向代理
echo ""
echo "[6/6] 配置 Nginx..."
sudo tee /etc/nginx/sites-available/report > /dev/null <<'EOF'
server {
    listen 80 default_server;
    
    client_max_body_size 50M;
    
    location / {
        proxy_pass http://127.0.0.1:8080;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
        proxy_read_timeout 300s;
    }
}
EOF

sudo rm -f /etc/nginx/sites-enabled/default
sudo ln -sf /etc/nginx/sites-available/report /etc/nginx/sites-enabled/
sudo nginx -t -q && sudo systemctl restart nginx
echo "✅ Nginx 配置完成"

# 输出结果
SERVER_IP=$(curl -s ifconfig.me)
echo ""
echo "============================================"
echo "🎉 部署完成！"
echo "============================================"
echo ""
echo "📡 访问地址："
echo "   本机:   http://localhost:8080"
echo "   公网:   http://$SERVER_IP"
echo ""
echo "⏰ 定时任务："
echo "   每天 8:20 (北京时间) 自动生成报表"
echo ""
echo "📋 管理命令："
echo "   查看协作日志:  tail -f $(pwd)/collab.log"
echo "   查看定时日志:  tail -f $(pwd)/cron.log"
echo "   重启协作服务:  sudo systemctl restart report-collab"
echo "   手动生成报表:  python3 更新点检表.py"
echo ""
echo "💡 提示：如需 HTTPS，请配置域名后运行："
echo "   sudo apt install -y certbot python3-certbot-nginx"
echo "   sudo certbot --nginx -d 你的域名"
echo "============================================"
```

使用方法：
```bash
chmod +x deploy.sh
./deploy.sh
```

## 🛠️ 常见问题

### Q: Oracle Cloud 免费实例会不会被回收？
A: 只要账号正常，**Always Free** 资源永久不会被回收。但 30 天试用期的付费资源会过期。

### Q: Render 免费版休眠了怎么办？
A: 可以用 https://cron-job.org 每 5 分钟 ping 一次你的应用地址，防止休眠。

### Q: 免费方案够用吗？
A: 完全够用。系统资源消耗很低（Python 进程 < 200MB 内存），免费方案都能轻松承载。

### Q: 数据安全吗？
A: 所有数据都在你自己的服务器/电脑上。Oracle Cloud 和 Cloudflare 都提供企业级安全防护。

### Q: 可以同时用多个方案吗？
A: 可以。比如用 Oracle Cloud 做主服务，用本地电脑做备份。
